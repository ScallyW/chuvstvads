# Discord Bot - Стейк Python + disnake + SQLite

Бот с тремя системами: заказы, отзывы, приветствия. Организован через коги для модульности.

## Установка

1. Установи зависимости:
```bash
pip install -r requirements.txt
```

2. Создай файл `.env` (скопируй из `.env.example`):
```bash
cp .env.example .env
```

3. Заполни `.env`:
```env
BOT_TOKEN=your_bot_token_here
ORDERS_CHANNEL_ID=123456789
REVIEWS_CHANNEL_ID=123456789
WELCOME_CHANNEL_ID=123456789
ORDER_PING_ROLE_IDS=123456789,987654321
WELCOME_ROLE_ID=123456789
GUILD_ID=123456789
```

4. Запусти бота:
```bash
python main.py
```

## Команды

### Текстовые (prefix: !)

| Команда | Права | Описание |
|---------|-------|----------|
| `!button_ord` | Админ | Отправить панель создания заказа |
| `!button_rew` | Админ | Отправить панель создания отзыва |
| `!welcome_setup #канал @роль [текст]` | Админ | Настроить приветствия |
| `!order_info <id>` | Все | Информация о заказе |

### Слеш-команды

- Бот автоматически синхронизирует слеш-команды при старте.

## Функции

### 1. Заказы (`!button_ord`)
- Модальное окно: описание, бюджет, сроки
- Отправка в указанный канал с пингом ролей из `ORDER_PING_ROLE_IDS`
- Сохранение в SQLite с ID заказа
- **Персистентная кнопка** — работает после перезапуска бота!

### 2. Отзывы (`!button_rew`)
- Модальное окно: оценка 1-5, текст
- Отправка красивого embed в канал отзывов
- Сохранение в SQLite
- **Персистентная кнопка** — работает после перезапуска бота!

### 3. Приветствия
- Автовыдача роли `WELCOME_ROLE_ID`
- Приветственное сообщение в `WELCOME_CHANNEL_ID`
- Кастомный шаблон через `{user}`, `{guild}`, `{member_count}`

## Персистентность кнопок

Кнопки используют:
- `timeout=None` — вью не истекает
- Фиксированный `custom_id` — идентификатор не меняется
- Регистрация в `__init__` кога через `bot.add_view()`

Это позволяет кнопкам работать даже после перезапуска бота, включая кнопки в старых сообщениях.

## Пример использования

1. Админ пишет `!button_ord` — отправляется embed с кнопкой
2. Пользователь нажимает кнопку — открывается модальное окно
3. Заполняет поля → заказ отправляется в канал + сохраняется в БД
4. Бот перезапускается...
5. Пользователь снова может нажать старую кнопку — она работает!
